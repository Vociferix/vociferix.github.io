Crasy - CoRoutine ASYnc

Crasy is a C++ async I/O library built on top of ASIO and based around C++20 coroutines. Additionally, Crasy is inspired by Rust's popular Tokio library. The intent is to enable developers to create "web scale" C++ applications while writing straight-line code.

Introduction

Crasy's two core types are executor and future. executor encapsulates the complete async runtime of Crasy, and future is the main building block of async coroutines that hook into Crasy's runtime. The bare minimum for a Crasy application is as follows:

#include <crasy/crasy.hpp>
crasy::future<int> async_main() {
co_return 0;
}
int main() {
return crasy::executor().block_on(async_main);
}

From here, a Crasy application consists of calls into various async I/O interfaces and task management utilities.
